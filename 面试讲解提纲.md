# AI 面试模拟官 — 面试讲解提纲

> 用途：面试时向面试官展示这个项目。核心策略——**讲过程，不讲完美代码**。
> 时长建议：3~5 分钟讲 + 现场演示 2 分钟。

---

## 开场（30 秒）

"我做了一个 AI 面试模拟官。简单说，就是把自己简历贴进去，AI 当面试官，针对我的简历逐轮追问、实时打分、指出我哪答得差，还能记录我每次的薄弱点、看进步曲线。我选这个项目，是因为它**三个优点叠在一起**：新手用 Python 标准库加免费大模型 API 就能做、拿来做作品展示面试官一眼懂、而且多轮追问和针对简历出题这种能力是**没有 AI 根本做不到的**。"

---

## 第一幕：选题 —— 为什么是它（1 分钟）

"课程给了 23 个场景，我筛掉了三类：难度高的新手做不完、太生活化的（穿搭饮食旅行，面试官看不到职业价值）、展示弱的。最后选面试模拟官，因为它新手能做、展示强、AI 核心价值高。这点我想强调——**选一个能讲清楚'为什么'的项目，比炫技重要**。"

---

## 第二幕：MVP —— 最小可用（45 秒）

"第一天我就出了 MVP：单文件，Python 标准库起一个网页服务，HTML 内嵌进去，一个端口。用户输入岗位和简历，AI 出题、用户答、AI 打分。技术上我定了个铁律——**内嵌单端口同源**，前端 fetch 走相对路径，这样用户拿到一个文件就能跑，没有前后端分离跨域白屏那堆破事。"

---

## 第三幕：痛点驱动迭代 1（1 分钟）

"然后我真实去用，用三份不同背景的简历做完整模拟——有实习丰富的、有跨行转产品的、有零实习应届生——挖出 6 个真实痛点。比如：
- 只有一种面试风格 → 我加了业务 / HR / 总监 / 压力四种面
- 面试前不诊断简历，第一轮就懵 → 加了面试前简历诊断
- 追问深度不可控 → 加了'深挖 / 换方向 / 跳过 / 结束'让用户掌控

每个痛点对应一次迭代，**这比直接写个完美系统更有说服力，因为我能讲清'我为什么这么改'**。"

---

## 第四幕：迭代 2 —— 历史追踪（含一个真实决策，1 分钟）

"第二个大迭代是历史追踪：每次面试自动记录岗位、类型、打分、弱点标签，下次面试前告诉我'你历史高频薄弱点是 X，重点准备'。

这里有个真实插曲——**我一度把它回滚了**。因为当时遇到一个白条问题，我误以为是历史功能导致的，为了保'干净'，就把功能删了。后来排查清楚，白条是预览面板内置浏览器在 localhost 存了凭据弹的自动填充，跟代码无关，删功能属于误伤。所以我**又把历史功能加回来了**，并且用'本机浏览器打开'绕开了白条。

我想用这个例子说明：**我会把问题定位到正确的层级，而不是在错误的方向上打转**——这比'没犯过错'更真实。"

---

## 第五幕：踩坑 —— 展示工程判断力（45 秒）

"除了白条，还有两个坑值得一提：
1. 历史弹窗'关闭'点不动——根因是 CSS 源顺序，`.modal` 的 `display:flex` 压过了 `.hidden` 的 `display:none`，两个优先级相同后定义的赢，一行 `!important` 根治。
2. 服务有时卡死变慢——根因是 `HTTPServer` 单线程，一次大模型调用阻塞全站，换成多线程 `ThreadingHTTPServer` 后并发请求零阻塞。

这两个都是**先用测试定位根因、再精准修，而不是拍脑袋**。"

---

## 第六幕：技术亮点速览（30 秒，备选）

- 单文件、零依赖、网页版，部署零成本
- 相对路径同源架构，根治跨域白屏
- 多线程 + 超时容错，LLM 长调用不卡死
- 弱点标签归一化（10 个标准维度），历史聚合可量化

---

## 收尾（20 秒）

"总结一下，这个项目最值得讲的不是代码有多花，而是**我怎么从用户真实痛点出发，一步步迭代，并且在踩坑时把问题定位到正确层级**。如果要现场看，我现在就能开起来跑一轮。"

---

## 演示建议（给自己）

1. **提前准备好一份真实简历文本**（避免现场打字卡壳）
2. **演示路径**：填岗位 → 诊断 → 业务面答 2 题（展示实时打分 + 弱点标签）→ 点「面试历史」看趋势
3. **应对刁钻问题**：
   - 被问"这跟套壳聊天机器人有什么区别" → 强调针对简历出题、弱点标签归一化、历史进步追踪，这些是产品化工程，不是裸调 API
   - 被问"为什么用相对路径 / 单端口" → 强调这是为了'一个文件就能跑、零部署成本'的用户体验决策
   - 被问"AI 偶尔输出不稳定怎么办" → 强调提示词约束 + 后端解析 + 超时容错三层兜底
