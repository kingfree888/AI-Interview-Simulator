# AI 面试模拟官 — 项目说明文档

> 适用场景：AI 个人系统实践课程 · 结题提交 / 面试作品展示
> 选题：课程场景清单 #8「AI 面试模拟官」（★★★ 零门槛）
> 版本：v2.2（含两次迭代 + 历史追踪闭环）
> 技术栈：Python 标准库 + DeepSeek 免费 API，单文件、零依赖、网页版

---

## 一、项目概述

AI 面试模拟官是一个**面向求职者的 AI 模拟面试练习工具**。用户输入目标岗位和自己的简历，AI 化身不同风格的面试官（业务 / HR / 总监 / 压力），针对简历逐轮提问、针对回答实时评价打分、指出薄弱点并给改进建议，面试结束后生成总评，并自动沉淀历史记录，帮用户追踪"哪些弱点反复出现、分数是否在进步"。

它解决的核心问题是：**真实面试机会贵、反馈慢、不知道自己差在哪**。用 AI 把"面试练习"变成随时可做的低门槛训练。

## 二、选题逻辑：为什么选 #8

课程提供了 23 个候选场景。我的筛选标准有三条：

1. **新手能做**：难度 < ★★，纯 Python 标准库 + 免费大模型 API，单文件 < 200 行
2. **展示强**：拿去面试，面试官一眼懂，还顺带证明"你既懂 AI、又懂面试这个真实场景"
3. **AI 核心价值高**：功能必须是"没有 AI 根本做不到"的，不能套壳

按这三条，我砍掉了三类：难度 ≥ ★★ 的、太生活化的（穿搭 / 饮食 / 旅行，面试官看不到职业价值）、展示弱的。最后留下 #8 面试模拟官——它三个优点叠在一起：新手能做、面试展示力强、AI 核心价值高（多轮追问、针对简历出题、实时打分，纯规则代码做不到）。

**对照课程"必做要求"**：
- 每个功能都体现 AI 核心价值：多轮追问、针对简历出题、弱点诊断——纯规则代码做不到 ✅
- 从清单选 1 个系统，1 天内出 MVP ✅
- 零门槛技术栈 ✅

## 三、技术栈与架构决策

**技术栈（刻意压到最低门槛）**
- 语言：Python 标准库（`http.server` / `urllib.request` / `json` / `re`），零第三方依赖
- 大模型：DeepSeek（`deepseek-chat`），学生有免费额度
- 形态：网页版（比终端更好演示），单文件 `interview_server.py`

**架构铁律：内嵌单端口同源**
这是实践中定下的关键决策：
- HTML / CSS / JS **全部内嵌**进 `interview_server.py` 这一个文件
- 后端用 `http.server` 单端口（8824）托管前端页面
- 前端 `fetch` 走**相对路径**（如 `/start`、`/action`），不写死 `http://localhost:8824`

好处：用户拿到一个文件，`python interview_server.py` 后浏览器打开即用，没有"前后端分离 → 跨域 → 白屏"这类部署灾难。

## 四、功能说明

### MVP（基础面试）
输入岗位 + 简历 → AI 面试官出题 → 用户作答 → AI 针对回答实时评价 / 打分 / 建议 → 下一题 → 面试总评。

### 迭代 1（AI 能力增强）
- **面试类型切换**：业务面（深挖项目）/ HR 面（软素质）/ 总监面（战略思维）/ 压力面（高压追问），不同风格不同追问策略
- **简历诊断分层**：
  - 空简历（< 80 字）：给"零经验应对策略"，不卡住最需要练习的人
  - 非空简历：给出针对性"弱点诊断"
- **追问控制**：深挖追问 / 换方向 / 跳过 / 结束，用户掌控节奏

### 迭代 2（个性化闭环 · 历史追踪）
- 每次面试结束自动写 `interview_history.json`（岗位 / 类型 / 轮次 / 平均分 / 弱点标签）
- 「面试历史」面板：高频薄弱点 Top6（进度条）、得分趋势、历史记录卡片
- 诊断页读取历史，提示"你历史高频薄弱点是 X（N 次），本次重点准备"

## 五、迭代过程：痛点驱动（核心亮点）

课程的评分点是"真实使用 + 痛点驱动迭代"。我的方法是：**先出 MVP → 真实用 → 挖痛点 → 改 → 再挖 → 再改**，每一步都留下记录。

我用 3 份不同背景的简历做了完整模拟测试（产品实习丰富的 A / 跨行转产品的 B / 零实习应届生 C），挖出 6 个真实痛点：
- **P2** 只有一种面试风格 → 迭代 1 加类型切换
- **P3** 面试前不诊断简历 → 迭代 1 加简历诊断
- **P4** 追问深度不可控 → 迭代 1 加追问控制
- **P5** 没有历史追踪 → 迭代 2 加历史闭环
- **P6** 空简历者得不到训练 → 简历诊断分层处理
- **P7** AI 偶尔不守输出格式 → 优化提示词 + 后端解析修复

每个痛点都对应一次迭代——这就是"为什么这么改"的过程，面试时能讲清楚。

## 六、遇到的关键问题与解决

### 问题 1：白条现象（展示渠道问题）
演示初期遇到预览面板里出现一条"白条"。排查结论：**它是预览面板内置浏览器在 localhost 存了凭据后弹的自动填充提示，与页面代码无关**（证据：把页面改成零表单的 contenteditable 纯 div，白条照样弹）。
**解法**：不纠缠代码，改用**自己本机浏览器（Chrome / Edge，或无痕模式）**打开 `localhost:8824`，干净无白条。这是"把问题定位到正确层级"的工程判断。

### 问题 2：弹窗关不掉（CSS 优先级 bug）
历史面板弹窗点了"关闭"没反应。根因是 CSS 源顺序：`.hidden{display:none}` 写在 `.modal{display:flex}` 之前，两个都是单类选择器、优先级相同，**后定义的 `.modal` 赢了**，导致 `hideHistory()` 加了 `hidden` 也关不掉。
**解法**：`.hidden{display:none!important}`，一行根治。

### 问题 3：响应慢 / 卡死（单线程阻塞）
`http.server.HTTPServer` 是**单线程**的，一次 LLM 调用（约数秒）会把整个服务堵死，浏览器所有请求排队超时。
**解法**：换成 `ThreadingHTTPServer`，实测面试进行中并发 10 个请求全部成功、零阻塞。同时给 LLM 调用加超时与错误捕获，单次失败不再卡死线程。

## 七、如何运行

```bash
# 1. 配置 DeepSeek API Key（interview_server.py 顶部 API_KEY）
# 2. 运行
python interview_server.py
# 3. 在自己本机浏览器地址栏打开（推荐无痕模式）
http://localhost:8824
```
> ⚠️ 不要用工具内的预览面板打开，那里会弹白条。
