# 使用建议与避坑

## 一、如何避开「白条」

**现象**：在工具的预览面板里打开 `localhost:8824` 时，页面上出现一条白色提示条，关不掉。

**根因**：这是预览面板内置浏览器在 localhost 源上保存了凭据后弹出的自动填充提示，**与本工具代码无关**。已验证：把页面改成零表单的 `contenteditable` 纯 div，白条照样弹——证明它不是本工具的问题。

**解法（铁律）**：**用你自己本机的 Chrome / Edge（推荐无痕模式 Ctrl+Shift+N）打开 `http://localhost:8824`**，干净无白条。不要在工具内的预览面板里看效果。

## 二、面试类型怎么选

| 类型 | 适合场景 | 追问风格 |
|---|---|---|
| 业务面 | 技术/产品岗，想深挖项目细节 | 追到答不上来为止，打分严格 |
| HR面 | 考察软素质、沟通、职业规划 | 温和，关注协作/抗压/价值观 |
| 总监面 | 中高级，考察商业思维 | 不抠执行，追问对业务的意义 |
| 压力面 | 想练抗压、临场反应 | 持续质疑，故意冷淡，打低分 |

**建议**：先 HR 面找感觉，再业务面练深度，最后压力面练心态。

## 三、如何用历史面板追踪进步

1. 用**同一份简历**连续跑 3~5 次面试（可混合类型）。
2. 每次结束自动写入 `interview_history.json`。
3. 点右上角「📊 面试历史」：
   - **高频薄弱点 Top6**：红色进度条，看哪些弱点反复出现（如「量化成果」总是低）。
   - **得分趋势**：色块看整体是否在涨。
   - **历史卡片**：回看每次的岗位/类型/平均分。
4. 下次面试前，诊断页会弹「你历史高频薄弱点是 X（N 次），本次重点准备」。

## 四、空简历怎么办

若简历 < 80 字（几乎空白），工具不会卡住，而是给「零经验应对策略」——告诉你没有简历时怎么准备面试、可以补充哪些方向让 AI 针对性出题。这是为最需要练习的应届生准备的兜底。

## 五、提交结果给别人用时

- 别人拿到 `skill/scripts/interview_server.py` 后，只需按 `references/config.md` 配好 DeepSeek Key 即可运行。
- 历史数据 `interview_history.json` 可删除重置（删除后自动重建为空）。
