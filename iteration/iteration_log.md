# 迭代升级说明（5步迭代法）

> 记录真实使用过程中遇到的痛点与改进。每次迭代按 **痛点 → 量化影响 → 假设原因 → 实现方案 → 评估效果** 五步记录。
> 共 3 次迭代（≥ 2 次要求）。

---

## 迭代 1：从「单一面试」到「分类型 + 简历诊断 + 追问控制」

**① 痛点**
初始 MVP 只有一种面试风格，无论什么岗位/简历都用同一套话术；面试前不诊断简历，第一轮就暴露问题；追问深度不可控（有时追问到底、有时草草跳过）。

**② 量化影响**
用 3 份不同背景简历（资深 / 转行 / 零经验）实测：用户练的是"通用面"，但真实面试场景差异巨大；用户常因简历漏洞在第一轮措手不及，信心受挫；想深挖的地方没深挖、想跳过的没跳过。

**③ 假设原因**
MVP 把"面试"简化为"聊天 + 打分"，没有建模真实面试的多维场景与节奏控制能力。

**④ 实现方案**
- 面试类型切换：业务面（深挖项目）/ HR面（软素质）/ 总监面（战略思维）/ 压力面（高压追问），各类型不同追问策略与打分尺度
- 面试前简历诊断分层：空简历（< 80 字）给「零经验应对策略」，非空给「弱点诊断」
- 追问控制按钮：深挖 / 换方向 / 跳过 / 结束，用户掌控节奏

**⑤ 评估效果**
用 3 份简历复测，AI 能针对不同岗位切换语气与深挖方向；简历诊断让用户在面试前知晓薄弱点；追问控制使练习更有针对性。对应 `痛点分析报告.md` 中 P2 / P3 / P4 闭环。

---

## 迭代 2：历史追踪闭环（含一次真实回滚决策）

**① 痛点**
每次面试完全独立，看不到"上次差在哪、这次是否进步、常犯什么错"，无法形成「发现弱点 → 专项练习 → 评估进步」的学习闭环。

**② 量化影响**
无历史时，用户重复踩同一个弱点（如"量化成果"）却无感知；产品价值停留在"单次练习"，缺乏留存与复访理由，弱化了 AI 个性化价值。

**③ 假设原因**
缺少持久化与聚合层，每轮打分 / 弱点标签用完即弃，无法跨场次比较。

**④ 实现方案**
- 后端 `/history` 端点 + `finish_record()` 写 `interview_history.json`（岗位 / 类型 / 轮次 / 平均分 / 弱点标签）
- 前端「面试历史」面板：高频薄弱点 Top6（进度条）、得分趋势、历史卡片
- 诊断页读历史，弹「你历史高频薄弱点是 X（N 次），本次重点准备」

**⑤ 评估效果（含真实决策）**
功能验证可用后，**一度因"白条"现象误判为历史功能导致而回滚移除**。后经排查：白条是预览面板内置浏览器在 localhost 存凭据后弹的自动填充，**与代码无关**（证据：把页面改成零表单的 `contenteditable` 纯 div，白条照样弹）。属误伤，遂**重新加回**历史功能，并用"本机浏览器打开 localhost"绕开白条。最终历史汇总 / 趋势 / 预警全部正常（见 `tests/test_record.md`）。这次"回滚再加回"成为展示工程判断力的真实案例——**把问题定位到正确层级，而非在错误方向打转**。

## 迭代 3（v2.2）：性能与流程根治

**① 痛点**
真实使用反馈：① 面试中无"发送回答"按钮；② 流程乱套（开场第一问就带打分）；③ 响应很慢 / 卡死。

**② 量化影响**
无发送键 → 用户不知如何提交；开场打分 → 未答先被评分，逻辑错乱；单线程服务一次 LLM 调用（约 5s）阻塞全站，浏览器所有请求排队超时（实测 `GET /` 都超时）。

**③ 假设原因**
- 无可见按钮（仅 Ctrl+Enter 快捷键）
- 提示词要求"每轮都打分"，开场即评分
- `http.server.HTTPServer` 是单线程，LLM 长调用阻塞整个服务

**④ 实现方案**
- 前端加「📤 发送回答」主按钮 + 提交时"思考中"提示，消除"点了没反应"错觉
- 提示词加 [流程铁律]：开场第一轮只提问不打分，之后每轮才给 `[评价][打分]X/10[建议][弱点][下一题]`
- 后端换 `ThreadingHTTPServer`；`call_deepseek` 加超时与 try/except 容错（单次失败不崩线程）；历史写加锁防并发损坏
- 修复弹窗关不掉：CSS `.hidden{display:none!important}`（根因是 `.modal{display:flex}` 源顺序压过 `.hidden`，两规则同优先级、后定义者赢）

**⑤ 评估效果**
实测：开场无打分、回答后才有评价 + 弱点；并发压测 10 个请求全部成功、零阻塞；弹窗可关。TC3 复测确认 HR 面开场亦无 `[评价]`。全部测试用例通过（见 `tests/test_record.md`）。
