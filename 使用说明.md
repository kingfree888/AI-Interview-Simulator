# AI 面试模拟官 · 使用说明

> 一个单文件、零第三方依赖的网页版 AI 模拟面试工具。输入岗位和简历，AI 化身不同风格的面试官逐轮追问、实时评价打分、指出薄弱点，并把每次面试沉淀成历史记录，帮你追踪进步。

---

## 一、这是什么

`interview-simulator` 是一个 **Hermes Skill**，核心是一个用 Python 标准库写成的单文件 Web 应用 `interview_server.py`。它调用 **DeepSeek** 大模型充当面试官，你在浏览器里跟它一问一答，完成一场完整的模拟面试。

- **技术栈极简**：只用 Python 标准库（`http.server` + `urllib`）+ DeepSeek API，不装任何 pip 包。
- **内嵌单端口同源**：HTML / CSS / JS 全部内嵌进一个 `.py` 文件，后端单端口 `8824` 同时托管前端和接口，没有跨域、没有前后端分离的部署烦恼。
- **开箱即用**：配好一个 API Key，`python` 一跑，浏览器打开就能面试。

## 二、核心功能

| 模块 | 能做什么 |
|---|---|
| **基础面试（MVP）** | 输入岗位 + 简历 → AI 出题 → 你作答 → AI 实时【评价 / 打分 / 建议】→ 下一题 → 面试总评 |
| **多类型面试官** | 业务面（深挖项目）/ HR 面（软素质）/ 总监面（战略思维）/ 压力面（高压追问）一键切换 |
| **简历诊断分层** | 空简历（<80 字）给「零经验应对策略」；有内容则给「弱点诊断」 |
| **节奏掌控** | 每题可选：深挖追问 / 换个方向 / 跳过 / 结束面试 |
| **历史追踪** | 面试结束自动写入 `interview_history.json`，「面试历史」面板展示高频薄弱点 Top6、得分趋势、历史卡片 |
| **弱点自适应** | 开新面试时自动读取历史 Top3 弱点注入面试官，**开场第一问就针对你的老毛病专项加练**，前端显示「🎯 针对性加练」徽章 |

## 三、快速开始（3 步）

### 第 1 步：准备一个 DeepSeek API Key（用你自己的额度）

本工具需要一个 DeepSeek 的 API Key 来调用大模型，费用走**你自己账户**的额度（新用户 / 学生通常有免费额度，个人练面试完全够用）。

1. 打开 <https://platform.deepseek.com/>
2. 用手机号注册 / 登录
3. 左侧菜单进入 **「API Keys」→「创建 API Key」**
4. 复制生成的 `sk-xxxxxxxx...`（**只显示一次，务必当场保存**）

### 第 2 步：把 Key 配置进工具（两种方式，任选其一）

#### ✅ 方式 A：设置环境变量（推荐，Key 不落进代码，最安全）

程序会自动从环境变量 `DEEPSEEK_API_KEY` 读取 Key。启动前先设置：

```powershell
# Windows PowerShell（当前这台机器）
$env:DEEPSEEK_API_KEY="sk-你自己的key"
python skill/scripts/interview_server.py
```

```bash
# Linux / macOS
export DEEPSEEK_API_KEY="sk-你自己的key"
python skill/scripts/interview_server.py
```

> 说明：这种方式设置的环境变量只在当前终端窗口有效，关掉窗口就失效，适合临时使用。想长期生效可在系统「环境变量」里永久添加一条 `DEEPSEEK_API_KEY`。

#### 方式 B：直接改代码（最简单，但注意别把 Key 传到公开仓库）

打开 `skill/scripts/interview_server.py`，找到文件顶部第 13 行左右：

```python
API_KEY = os.environ.get("DEEPSEEK_API_KEY", "")  # 也可直接填 "sk-你的key"
```

把它改成你自己的 Key（**保留引号**）：

```python
API_KEY = "sk-你自己的key"
```

> ⚠️ **安全提醒**：如果你要把代码上传到 GitHub 等公开仓库，**务必改回方式 A 或提交前清空 Key**，绝不要把真实 Key 推上去，否则可能被盗刷。

### 第 3 步：运行并打开

```bash
python skill/scripts/interview_server.py
```

启动成功后，**用你自己电脑的浏览器**（Chrome / Edge，推荐无痕窗口）在地址栏打开：

```
http://localhost:8824
```

填入目标岗位 + 简历，点「诊断简历，开始面试」即可开始。

> ⚠️ **不要用工具内置的预览面板打开 localhost 链接**。预览面板的内置浏览器会因凭据自动填充弹出「白条」遮挡页面——这是浏览器行为，与本工具代码无关。用你本机的 Chrome / Edge 打开就是干净的。

## 四、可调参数

打开 `skill/scripts/interview_server.py` 顶部即可修改：

| 参数 | 默认值 | 说明 |
|---|---|---|
| `API_KEY` | 读环境变量 | 你的 DeepSeek Key |
| `MODEL` | `deepseek-chat` | 使用的模型，可换成其它 DeepSeek 模型 |
| `PORT` | `8824` | 服务端口，被占用时改成别的（如 `8825`），浏览器地址同步改 |
| `temperature` | `0.6`（在 `call_deepseek()` 内） | 越高回答越发散，越低越稳定 |

## 五、验证是否配置成功

1. 运行后终端没有报错，提示服务已在 `8824` 启动。
2. 浏览器打开 `http://localhost:8824` 能看到页面。
3. 填岗位 + 简历点开始，能正常收到面试官的第一个提问 → **Key 生效**。
4. 若提示「调用失败」：
   - 检查 Key 是否复制完整、有没有多余空格；
   - 检查网络能否访问 DeepSeek（`api.deepseek.com`）；
   - 检查 DeepSeek 账户是否还有额度。

## 六、常见问题（FAQ）

**Q：一定要用 DeepSeek 吗？能用别的模型吗？**
A：默认对接 DeepSeek。想换其它兼容 OpenAI 格式的服务，需要改 `call_deepseek()` 里的请求地址和 `MODEL`。

**Q：会花很多钱吗？**
A：`deepseek-chat` 价格很低，一场几十轮的面试消耗极小，新用户免费额度通常够用。费用走你自己账户。

**Q：页面打不开 / 打开是白条？**
A：确认①程序还在运行（终端没关）；②用的是**本机浏览器**而非预览面板；③端口没被占用（可改 `PORT`）。

**Q：历史数据存在哪？**
A：程序运行目录下的 `interview_history.json`，删除它即清空所有历史。

**Q：端口 8824 被占用怎么办？**
A：改 `interview_server.py` 里的 `PORT`，并用新端口打开浏览器。
